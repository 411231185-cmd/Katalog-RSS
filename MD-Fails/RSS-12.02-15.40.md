# Для автоматизации поиска и генерации посадочных страниц мы будем работать с файлом data/normalized/catalog_with_tilda_urls.csv. Он самый насыщенный, свежий и максимально соответствует структуре Tilda.

Принято: `data/normalized/catalog_with_tilda_urls.csv` — хороший “источник истины” для сравнения, потому что в нём уже есть готовые URL.

## Что важно в этом CSV

- Разделитель `;` (не запятая), это критично для корректного чтения.
- В шапке есть колонка **Url** (в примерах это ссылки вида `http://project17325576.tilda.ws/tproduct/...-slug`).
- Есть все базовые поля для отчёта и генерации: `SKU`, `Title`, `Category`, `Photo`, `Price`, `Tilda UID/UID`.
- В данных встречаются дубли по SKU/Title (в фрагменте один и тот же SKU идёт несколькими строками), значит скрипт должен уметь: “dedupe” и отчёт по коллизиям.


## Как правильно сравнивать с sitemap.xml

Чтобы не “убиться” доменами (у вас в каталоге — `project17325576.tilda.ws`, а на сайте может быть `tdrusstankosbyt.ru`), сравниваем не целиком URL, а **slug** (последний сегмент пути).

Алгоритм сравнения:

- Из sitemap берём все `<loc>` URL → нормализуем → `slug`.
- Из каталога берём поле `Url` → нормализуем → `slug`.
- Missing = те строки каталога, чей `slug` отсутствует в sitemap.
- Отдельно считаем:
- `empty_url_in_catalog` (Url пустой/битый)
- `duplicate_slug_in_catalog` (коллизии после нормализации)
- `duplicate_sku` (один SKU в нескольких строках)

Нормализация URL→slug:

- Убрать `?query` и `#anchor`
- Убрать завершающий `/`
- Взять последний сегмент пути
- Привести к lower-case

## Проверка Python-кода

Ваш скрипт — рабочая база, но требует двух важных дополнений для стабильности:[^8_1]

- **Разделители в CSV:** В ваших файлах (например, `catalog_with_tilda_urls.csv`) часто используется `;`, а `read_csv` по умолчанию ищет `,`. Нужно добавить `sep=None, engine='python'`, чтобы pandas сам определил разделитель.
- **Кодировка:** Для кириллицы лучше явно указывать `encoding='utf-8-sig'` (если файл с BOM) или `cp1251`.[^8_2]
- **Формат .xls:** Приложенный вами файл `1.xls` успешно читается, но содержит 137 строк с неразмеченными колонками (`Unnamed`). Это значит, что при автоматической сборке ТКП скрипт должен будет искать данные по ключевым словам в тексте, а не по фиксированным заголовкам.[^8_1]
