# Katalog-RSS — контекст проекта

## Назначение
- Генерация и поддержка каталога RSS для ТД «РУССтанкоСбыт».
- Синхронизация номенклатуры, офферов и атрибутов для сайта и маркетплейсов.
- Подготовка данных для рекламы (Яндекс.Директ, РСЯ, возможно другие каналы).

## Текущие задачи для Claude
- Рефакторинг скриптов обновления каталога (Python/PowerShell).
- Автоматизация очистки, нормализации и дедупликации товарных файлов (CSV/MD/JSON).
- Генерация и поддержка вспомогательных скриптов (парсеры, миграции, отчёты).
- Подготовка агентов для связки: Katalog-RSS ↔ Яндекс.Директ ↔ Dashboard-RSS.

## Структура проекта (упростить, если нужно)
- `Товары/` — выгрузки, офферы, каталоги.
- `scripts/` или `ps1/` — скрипты на PowerShell / Python.
- `config/` — настройки, маппинги полей, шаблоны.
- `docs/` — описания форматов, требования площадок.

## Правила работы для Claude
- Отвечай на русском.
- Перед изменением файла — прочитай файл и кратко перескажи, что он делает.
- Не создавай временных файлов в корне проекта; если нужно — используй `tmp/` или `sandbox/`.
- Не меняй формат входных файлов без явного запроса (особенно CSV/MD, которые уже используются на сайте/в рекламе).
- Предлагай миграционный план, если меняешь структуру данных.
- Не выполняй git-коммиты и не меняй CI/CD без моей явной команды.

## Предпочтения по коду
- Python: чистые функции, явные типы там, где помогает, минимум магии.
- PowerShell: максимально читаемые команды, комментарии только к нетривиальным шагам.
- Скрипты должны уметь работать из корня репозитория без сложной ручной подготовки.
